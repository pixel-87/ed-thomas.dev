Like how git began tracking it's own development after around a day, I want to follow in the footsteps of the greats, by making my first blog post about how I built this blog.
Maybe this will serve as a guide to someone one day, or just a way for me to get my ideas actually written (or typed) down.

# Why make a blog
I think it would be fun to be able to carve out this little section of the internet however I like, not just a personalised description or posts, but entirely how I want it to be. 
I've seen some incredible looking sites out there, from some really talented people, and maybe if one day someone can look at my site and think the same thing, leaving that lasting impression would be worth all the effort.

Also blogs are meant to be good for recruiters, so two birds one stone.

# Tech stack
## Static Site Generator (SSG) - Hugo
I'm not well versed in the world of SSGs, but have seen Hugo being recommend before, and there's no better way of finding out its pros and cons than just using it.

From what I've read, it is simple to use and fast, which sounds perfect.

I'm very familiar using markdown in my obsidian notes so it makes sense to continue using it.

## Hosting - Self-hosted
I've already got a few things self-hosted like pihole and a minecraft server, so I don't really see the point in handing off the responsibility to another party.
My site doesn't need 100% uptime, won't get that much traffic, so doing everything myself is completely reasonable, and makes for a better learning experience. 

## Web Server - Caddy
While I could of gone for a classic, battle-tested web server like Nginx, Caddy is easier to get up and running, and handles things like HTTPS automatically out of the box.
